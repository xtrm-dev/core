## Late-Day Addendum — Worktree `.beads` Symlink Removal (post-compact)

After the initial 2026-05-12 release-prep work above, the operator surfaced a real incident from `projects/infra` PR #39: a worktree-internal `.beads` symlink was baked into `main` via squash-merge, wiping the entire `.beads/` directory from git (recovered via PR #40). This exposed that the existing two-layer defense (suppress-noise at provisioning + `xt end` pre-push guard) had a structural gap: the symlink itself was the hazard, not just the diff noise.

### Investigation

Reasoned about why `.beads/` is provisioned into a worktree at all when bd uses dolt shared-server. Discovered via `bd worktree create --help` and empirical check that **modern bd 1.0.3 stores `core.hooksPath` as an ABSOLUTE parent path at `bd init` time** — meaning every worktree inherits parent hooks via shared git config without needing per-worktree config tweaks. Audited `.beads/hooks/*` and confirmed they are pure `bd hooks run` delegators with no `mkdir`/scaffold logic.

### Prototype

Tested the simplified approach in `/tmp/bd-proto` (fresh embedded bd repo) and then on the **real xtrm-tools shared-server worktree** (`/tmp/xt-wt-proto`, branch `xt-wt-proto`):

| Step | Result |
|------|--------|
| `bd worktree create /tmp/xt-wt-proto` | Standard worktree, `.beads/` checked out (10 tracked files) |
| `rm -rf .beads` + `git update-index --skip-worktree -- $(git ls-files .beads)` | `git status -s` clean |
| `bd ready` from inside worktree | Returns real xtrm-tools shared-server beads — confirms common-dir resolution works without local `.beads/` |
| `bd create` (filed throwaway `xtrm-v8ke`) | Wrote to shared server; no `.beads/` reappeared |
| `git commit` of `proto-marker.txt` | Clean commit; no pre/post-commit hook re-scaffolded `.beads/` |
| **`git diff main..HEAD --raw`** | `:000000 100644 ... A proto-marker.txt` — **zero `.beads/*` entries, zero 120000 modes** |
| `bd kv set` + `bd close` (memory gate lifecycle) | All work end-to-end |
| `git worktree remove --force` + `bd close xtrm-v8ke` | Clean cleanup |
