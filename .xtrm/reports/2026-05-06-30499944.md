## Code Changes

### Pi extension runtime

`packages/pi-extensions/extensions/xtrm-ui/index.ts` now adds a `hideThinkingPlaceholder` preference defaulting to `false`. On refresh, xtrm-ui calls `ctx.ui.setHiddenThinkingLabel("")` so Pi's hidden-thinking state has no label text. Because Pi core would otherwise still render an empty text row, xtrm-ui patches `AssistantMessageComponent.updateContent()` at runtime: when the hidden-thinking label is empty and the component is not expanded, `thinking` content blocks are filtered out before render; when expanded via the standard `Ctrl+O` tool-output expansion path, the patch temporarily renders thinking as visible so full traces are shown. The patch also adds a `setExpanded()` method to assistant messages so Pi's existing expansion loop can control thinking visibility consistently with tools.

### Package release metadata

`packages/pi-extensions/package.json` was bumped to `0.7.17`, and `package-lock.json` was refreshed so the workspace lock reflects the published pi-extensions version. `npm publish --workspace @jaggerxtrm/pi-extensions --tag latest --access public` succeeded; `npm view @jaggerxtrm/pi-extensions version` returns `0.7.17`.
