## Code Changes

### Hook and policy changes

- Added `.xtrm/hooks/specialists-agent-guard.mjs`, a Claude PreToolUse hook that reads hook input, checks only `Agent` calls, detects `using-specialists` / `using-specialists-v2` markers in direct prompt fields or the transcript tail, and emits a compact JSON block decision: `Use specialists CLI instead of Agent tool. Route via: specialists run <name> --bead <id>`.
- Added `policies/specialists-agent-guard.json` with `runtime: claude`, `PreToolUse`, matcher `Agent`, and a 2s timeout.
- Recompiled `.xtrm/config/hooks.json` from policies and regenerated `.xtrm/registry.json`, so installs receive the new hook and registry metadata.

### Pi runtime release packaging

- Updated root `package.json` `files` to include `scripts/patch-external-pi-tools.mjs`. The runtime already calls `runExternalPiToolPatch()` from `runPiRuntimeSync`; this change ensures the script is actually present in the published tarball.
- Rebuilt `cli/dist/index.cjs` and source map with `npm run build --workspace cli` / `npm run prepublishOnly`.

### Pi UX fixes

- Removed the noisy xtrm-loader `.xtrm/memory.md` injection INFO log from `.xtrm/packages/pi-extensions/extensions/xtrm-loader/index.ts` while preserving system-prompt injection.
- Changed packaged dark xtrm-ui `toolPendingBg` from `surfaceAlt` to `surfaceMuted` to reduce overly black row background contrast.

### Tests

- Updated `cli/test/pi-runtime.test.ts` so the install-runner mock returns `{ status: 0, stdout: '', stderr: '' }`, matching the current `PiPackageInstallRunner` contract.
