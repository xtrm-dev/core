## Problems Encountered

| Problem | Root Cause | Resolution |
|---------|------------|------------|
| Project-local Pi package shadowed global reinstall; `/reload` still showed old xtrm-ui behavior. | Pi loaded `.pi/npm/node_modules/@jaggerxtrm/pi-extensions` version `0.7.17` while the global symlink pointed at updated code. | Reinstalled both global and project-local package copies with npm. |
| External frame patch initially modified the wrong Pi prototype. | `import.meta.resolve('@mariozechner/pi-coding-agent')` resolved a stale `~/node_modules` copy instead of the live `@earendil-works/pi-coding-agent` package. | Added live Pi entrypoint resolution using `process.argv[1]` / global `@earendil-works` path before legacy fallback. |
| `/reload` kept old border-label renderer. | Prototype patch used a boolean sentinel, so newer closures did not replace older patched renderers. | Versioned the patch sentinel so reloads replace older implementations. |
| Compact external rows duplicated tool call and result text. | Renderer framed Pi's native call/result output when compact result content was not yet available. | Compact mode now renders only result content or a one-line pending summary from tool args. |
| Ctrl+O expanded view did not show full output. | `tool_result` compaction replaced the original content with a summary. | Original text is stored as `details.xtrmOriginalText`; expanded render uses that text. |
| One-line pseudo-frame looked visually broken. | Underline/overline ANSI styling was not portable in Pi/TUI and appeared to erase text. | Replaced pseudo-frame experiment with selectable modes; default is background row, optional `box` mode keeps real box drawing. |
| Pi crashed with `Rendered line exceeds terminal width`. | Vibrant badge styling added visible spaces after row truncation. | Badge now styles the matched prefix in-place with no added visible padding. |
| `sp-terminal-overlay` crashed with handle errors. | Initial overlay code scoped `handle` incorrectly and assumed `onHandle` exposed `requestRender()` in the current Pi runtime. | Removed handle usage and use documented `tui.requestRender()` only. |
| `sp-terminal-overlay` risked destabilizing Pi during live streams. | It requested full redraws on every output chunk and double-closed. | Added 100ms render throttling, timer cleanup on dispose, and a single close path. |
| `sp ps --follow` accumulated repaint frames. | Output parser treated cursor/clear ANSI sequences as append-only text. | Added conservative CSI handling for cursor home, clear screen/line, and cursor movement. |
| `sp feed` / `sp ps` colors initially rendered plain white. | SGR color sequences were consumed and discarded. | Preserved allowlisted numeric SGR in append-only mode, reset per row, and stripped SGR in cursor-addressed screen mode per overthinker recommendation. |
| GitHub #257 looked like a request for xtrm-managed dependency provisioning. | Git worktrees are clean checkouts and dependency setup is project-specific. | Chose minimal UX/docs: explain clean-worktree behavior and point users/specialists to standard repo bootstrap commands. |

## Code Changes

### xtrm-ui external tool chrome

- Added compact summaries for external tools with tailored handling for Serena, GitNexus, `structured_return`, `process`, and generic extension tools.
- Patched Pi `ToolExecutionComponent` fail-open so non-native tools can render in XTRM's compact density while native tool rendering remains unchanged.
- Added `externalToolChrome` preference with `background` default and `box` optional, plus `/xtrm-ui chrome background|box` and `/xtrm-ui-external-chrome background|box` commands.
- Added vibrant badge highlighting for the compact tool-name prefix without changing visible row width.
- Preserved expanded-output behavior through `details.xtrmOriginalText`.

### sp-terminal-overlay extension

- Added managed extension `packages/pi-extensions/extensions/sp-terminal-overlay/` and registered it through `packages/pi-extensions/src/extensions/sp-terminal-overlay.ts` and `src/registry.ts`.
- Added `/sp-feed [args]`, `/sp-ps [args]`, `/xtrm-ps [args]`, and `/xtrm-terminal <command>`.
- Implemented centered 80% overlay with fixed 24-row output body, scroll controls, restart/close keys, bounded buffer, redraw throttling, process cleanup, and conservative ANSI handling.

### #257 worktree bootstrap guidance

- `cli/src/utils/worktree-session.ts`: staged change in the `xtrm-tbih` worktree adds a post-create note that clean worktrees omit ignored dependency dirs and tells users to run their repo bootstrap if lint/tests need dependencies.
- `using-xtrm` and `using-specialists-v3`: staged guidance says specialist worktrees are clean git checkouts, missing tools should be handled via repo bootstrap, and dependency artifacts should not be tracked.
- Specialists canonical `config/skills/using-specialists-v3/SKILL.md`: staged matching guidance in `/home/dawid/dev/specialists`.
