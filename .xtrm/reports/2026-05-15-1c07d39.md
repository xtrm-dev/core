## Problems Encountered

| Problem | Root Cause | Resolution |
|---------|-----------|------------|
| Project-local Pi package shadowed global reinstall; `/reload` still showed old xtrm-ui behavior. | Pi loaded `.pi/npm/node_modules/@jaggerxtrm/pi-extensions` version `0.7.17` while the global symlink pointed at updated code. | Reinstalled both global and project-local package copies with `npm install -g ./packages/pi-extensions` and `npm install --prefix .pi/npm ./packages/pi-extensions`. |
| External frame patch initially modified the wrong Pi prototype. | `import.meta.resolve('@mariozechner/pi-coding-agent')` resolved a stale `~/node_modules` copy instead of the live `@earendil-works/pi-coding-agent` package. | Added live Pi entrypoint resolution using `process.argv[1]` / global `@earendil-works` path before legacy fallback. |
| `/reload` kept old border-label renderer. | Prototype patch used a boolean sentinel, so newer closures did not replace older patched renderers. | Versioned the patch sentinel so reloads replace older patch implementations. |
| Compact external rows duplicated tool call and result text. | Renderer framed Pi's native call/result output when compact result content was not yet available. | Compact mode now renders only result content or a one-line pending summary from tool args. |
| Ctrl+O expanded view did not show full output. | `tool_result` compaction replaced the original content with a summary. | Original text is stored as `details.xtrmOriginalText`; expanded render uses that text. |
| One-line pseudo-frame looked visually broken. | Underline/overline ANSI styling was not portable in Pi/TUI and appeared to erase text. | Replaced pseudo-frame experiment with selectable modes; default is background row, optional `box` mode keeps real box drawing. |
| Pi crashed with `Rendered line exceeds terminal width (58 > 56)`. | Vibrant badge styling added visible spaces after row truncation, increasing the rendered width by two columns. | Badge now styles the matched prefix in-place with no added visible padding; patch bumped to v10. |
| `sp-terminal-overlay` crashed with `handle is not defined`, then `handle?.requestRender is not a function`. | Initial overlay code scoped `handle` incorrectly and assumed `onHandle` exposed `requestRender()` in the current Pi runtime. | Removed handle usage entirely and use documented `tui.requestRender()` only. |
| `sp-terminal-overlay` risked destabilizing Pi during live streams. | It requested full redraws on every output chunk and double-closed with both `handle.close()` and `done()`. | Added 100ms render throttling, timer cleanup on dispose, and a single close path. |
| `sp ps --follow` accumulated repaint frames. | Output parser treated cursor/clear ANSI sequences as plain append-only output. | Added conservative CSI handling for cursor home, clear screen/line, and cursor movement to emulate repaint dashboards. |
| `sp feed` / `sp ps` colors initially rendered plain white. | SGR color sequences (`CSI ... m`) were consumed and discarded. | Preserved allowlisted numeric SGR in append-only mode, reset per rendered row, and stripped SGR in cursor-addressed screen mode per overthinker recommendation. |
| Cleanup found pre-existing worktrees, tmux sessions, Serena/GitNexus MCP servers, Dolt warning, and `/tmp/sp-*` dirs. | These were not created by this session (existing xt worktrees, attached `specialists` tmux, MCP servers, older script temp dirs). | Session-created specialist jobs/tmux sessions were stopped; pre-existing artifacts were left intact and documented here. `sp ps` is clean for this branch. |

## Code Changes

### `xtrm-ui` external tool chrome

- Added compact summaries for external tools with tailored handling for Serena, GitNexus, `structured_return`, `process`, and generic extension tools.
- Patched Pi `ToolExecutionComponent` fail-open so non-native tools can render in XTRM's compact density while native tool rendering remains unchanged.
- Added `externalToolChrome` preference (`background` default, `box` optional), persistence, status reporting, and slash commands:
  - `/xtrm-ui chrome background|box`
  - `/xtrm-ui-external-chrome background|box`
  - existing compact/density commands continue to work.
- Added vibrant badge highlighting for the compact tool-name prefix without changing visible row width.
- Preserved expanded-output behavior through `details.xtrmOriginalText`.

### `sp-terminal-overlay` extension

- Added a new managed extension at `packages/pi-extensions/extensions/sp-terminal-overlay/` and registered it via `packages/pi-extensions/src/extensions/sp-terminal-overlay.ts` + `src/registry.ts`.
- Added commands:
  - `/sp-feed [args]` → `sp feed -f [args]`
  - `/sp-ps [args]` / `/xtrm-ps [args]` → `sp ps --follow` by default
  - `/xtrm-terminal <command>` → arbitrary shell command stream
- Implemented centered 80% overlay with fixed 24-row output body, scroll controls, restart/close keys, bounded buffer, redraw throttling, process cleanup, and conservative ANSI handling.

### Runtime/package wiring

- Reinstalled `@jaggerxtrm/pi-extensions` globally and into project-local `.pi/npm` repeatedly during validation because Pi can load either source depending on project settings.
