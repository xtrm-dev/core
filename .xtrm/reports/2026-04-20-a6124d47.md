# Session Report — 2026-04-20

## Summary

Removed the `active/pi` + `active/claude` runtime split from the xtrm skills materialization layer, collapsing both into a single flat `active/` view. The split was a historical artifact from when pi needed to exclude gitnexus skills (npm-provided); that constraint no longer applies. The fix spans `skills-layout.ts`, `skills-materializer.ts`, `skills-scaffold.ts`, `pi-runtime.ts`, `skills-runtime-views.ts`, `init-verification.ts`, `worktree-session.ts`, and associated tests — one commit via specialist executor, one fix commit after reviewer flagged that `rebuildAllRuntimeActiveViews` was using only claude's `enabledPacks` (fixed by taking the union of claude + pi pack sets). Post-merge, the dist hadn't been rebuilt so `xt init` was verifying against stale compiled code and reporting false failures; rebuilt and re-ran to confirm all 5 phases green. Wrote `update-xt` skill — a canonical-state reconciler that embeds the current install state as an explicit detection + fix + verify table, validated live against this project which was itself drifted after the merge. Bumped and published 0.7.11 (0.7.10 was already taken). Filed `unitAI-rztq7` P0 in `dev/specialists` to build the equivalent reconciler for the specialists runtime. Killed two `sp feed -f` processes that had been running 27h+ consuming ~62% CPU combined.
