## Code Changes

### Change summary
- Added: 19+ files across CLI commands/tests/docs/skills/report helper surfaces.
- Modified: 30+ files including core skills runtime, install/update/doctor/release command registration, registry, docs, and bundled skill/runtime metadata.
- Deleted: intentionally removed stale/static specialists-owned skill copies from repo-at-rest; vendor flow rehydrates them for publish from `SPECIALISTS_REPO_PATH`.

### Cat B distribution and runtime layout
- `cli/src/core/skills-layout.ts`, `skills-materializer.ts`, `skills-state.ts`, `skills-scaffold.ts`, `pi-runtime.ts`, and related tests were updated around the flat skills model. `.xtrm/skills/active/` is the single runtime view; per-runtime `active/claude` and `active/pi` directories are not part of the model.
- `.pi/settings.json` now points only at `../.xtrm/skills/active`.
- `scripts/vendor-specialists-skills.mjs` and `package.json` prepublish wiring make xtrm-tools the distributor while leaving seven skills canonically authored in the specialists repo.

### Doctor and update surfaces
- `cli/src/commands/doctor.ts` now includes Cat B drift status, runtime view checks, duplicate canonical-name guard, human output, `--json`, and `--check-drift` behavior. `cli/test/doctor.test.ts` covers clean/drift cases.
- `cli/src/commands/update.ts` and `cli/src/core/repo-discovery.ts` add `xt update` dry-run/apply behavior for one repo or all xtrm-managed repos under a root, with JSON/human output and failure aggregation. `cli/src/tests/update.test.ts` covers the command behavior.

### Migration and documentation
- `docs/cat-b-distribution.md`, `docs/skills-ownership.md`, README/AGENTS/CLAUDE references, and migration tests document the ownership model, prepublish vendor flow, migration safety around existing `.claude/skills`, and Windows posture.

### Release command
- `cli/src/commands/release.ts` adds `xt release prepare` and `xt release publish`. `prepare` invokes `sp script changelog-keeper` in read-only template mode; `publish` handles tag/push and optional GitHub release flow.
- `cli/src/core/xt-reports.ts` is now the canonical report bundle helper. It uses `git log -1 --format=%cs` for annotated tag dates.
- `.xtrm/skills/default/releasing/scripts/xt-reports.ts` is a thin wrapper to the canonical helper, preserving the releasing skill surface.
- `cli/src/tests/release.test.ts` covers annotated tags, non-empty report bundles, prepare output shape, tag/version derivation, and release scope guards for staged/unstaged stray files.
