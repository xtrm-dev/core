# Session Report — 2026-04-02

## Summary

Six init/runtime bugs fixed across hooks path generation, MCP sync behavior, skills runtime verification, MCP filename canonicalization, report bloat, and a deep skills materializer corruption issue. The most significant discovery was a two-layer skills runtime verification failure: (1) `registry.json` contained a `quality-gates/.claude/skills/using-quality-gates/SKILL.md` entry that triggered a `NESTED_RUNTIME_ROOT` invariant abort in `ensureAgentsSkillsSymlink`, silently preventing `active/claude` from ever being populated on fresh init; and (2) `hasOnlyValidSymlinkEntries` rejects any real directory in `active/claude/` — writes through the `.claude/skills` symlink from agent tooling create real dirs that survive the atomic swap check. The debugger spent 15 minutes tracing call chains across `skills-materializer.ts`, `skills-runtime-views.ts`, `init-verification.ts`, and `init.ts` before pinpointing both causes. All other issues were single-wave executor dispatches: `claude-runtime-sync.ts` was writing `${CLAUDE_PLUGIN_ROOT}/hooks/` paths that break when the env var is unset; init's MCP sync was overwriting `.mcp.json` clobbering user customizations; `mcp_servers.json` was renamed to `claude.mcp.json` for parity with `pi.mcp.json`; and `xt report generate` was emitting full git diff blocks that inflated handoff token count and had a broken specialists section.

## Code Changes

### CLI core (init/sync pipeline)
- **`claude-runtime-sync.ts`**: Added `rewritePluginRootCommandToProjectHookPath()` — purges `${CLAUDE_PLUGIN_ROOT}` from generated hook commands, writes absolute `.xtrm/hooks/` paths.
- **`project-mcp-sync.ts`**: `preserveExistingFile` option — init merges canonical servers into existing `.mcp.json` rather than overwriting. User servers survive re-runs.
- **`init.ts`**: MCP config references updated to `claude.mcp.json`; `service-registry.json` `skill_path` now targets `.xtrm/skills/default/`; Phase 6b activation notice added.
- **`init-verification.ts`**: Hooks check normalizes path separators; checks project and global `settings.json`; returns success on first valid xtrm wiring found.

### Skills runtime
- **`skills-materializer.ts`**: Detects non-symlink entries in `active/<runtime>` before `atomicSwapDirectory` and emits named eviction warning. Prevents silent corruption accumulation.
- **`skills-scaffold.ts`**: `.claude/skills` guarded as read-only when it exists as a real path. Activation counts returned from `ensureAgentsSkillsSymlink`.

### Config / registry
- **`.xtrm/registry.json`**: Removed `quality-gates/.claude/skills/using-quality-gates/SKILL.md` (NESTED_RUNTIME_ROOT fix). Renamed MCP config entries. Refreshed stale hashes.
- **`.xtrm/config/`**: `mcp_servers.json` → `claude.mcp.json`, `mcp_servers_optional.json` → `claude.mcp.optional.json`.

### Report CLI
- **`report.ts`**: Removed `git diff --stat` and diff template block. Change summary is now compact counts + capped file list. Specialists section rebuilt from `status.json` with session start filter, capped at 25 rows.

### Tests
