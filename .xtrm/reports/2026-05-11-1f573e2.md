## Code Changes

### Headline modifications

- **`cli/src/commands/docs.ts`** (C1): replaced `findRepoRoot()` with `findProjectRoot()` in `show`, `list`, `cross-check`, `verify` — docs scanner now respects cwd / fixture boundaries instead of always traversing the package source's `docs/` (22 files).
- **`cli/test/extensions/*` + `extension-harness.ts`** (C2): aligned test mocks with current runtime extension contracts — corrected import path (`packages/pi-extensions/src/core/lib` → `src/core`), added `sessionManager.getSessionId()` mock, updated custom-footer's `bd kv` subprocess stub to the current `bd list --status=in_progress` parsing path.
- **`policies/beads.json` + `scripts/compile-policies.mjs` + `.xtrm/config/hooks.json`** (C3): added beads-compact-save / beads-compact-restore hook entries via the policy source (not direct hooks.json edits — which had broken compile-policies golden and runtime). Compiler now emits a wrapper-level `script` field for beads-compact wrappers only.
- **`cli/src/commands/{init,update}.ts`** (C4): `xtrm update --help` now mentions the init alias; `runProjectInit` wraps `findRepoRoot` with a `Compilation failed` throw when the source root can't be resolved.
- **`scripts/gen-registry.mjs`** + **`.xtrm/registry.json`** (xtrm-xvjg): dropped `pi_extensions` asset entry; project scaffold no longer copies `packages/pi-extensions` into project `.xtrm/` (global-only install policy).
- **`cli/src/tests/registry.test.ts`**: idempotence-test asset keys reverted to `['config', 'hooks', 'skills', 'skills_optional']` after operator confirmation that pi-extensions are not per-repo.
- **`.beads/`** (prerequisite via `xtrm-f3s2`): restored as tracked directory from `e3a260b` (last good pre-#196 state); sensitive runtime files (`.beads-credential-key`, `interactions.jsonl`) excluded from tracking; `dolt.shared-server: true` added to `.beads/config.yaml`.
