## Problems Encountered

| Problem | Root Cause | Resolution |
|---------|-----------|------------|
| `agent-docs-maintainer` missed repos that start with managed boilerplate | The audit only counted global lines, code fences, command refs, broad bloat headings, and hardcoded stale terms; it did not ask whether the top of the file explained what the repo is. | Added `detect_repo_identity()` and markdown/json output for identity status, including a hard recommendation when the first content is a managed block or there is no H1/H2/prose before managed blocks. |
| Useful Stack Overview prose was treated as generic bloat | The previous 300/500 line thresholds applied to total file length, so a deliberate 100+ line repo overview triggered trim advice even when routing boilerplate was compact. | Added split size metrics (`repo_identity_lines`, `routing_lines`, `managed_lines`) and changed recommendations to apply caps to routing/managed content while preserving substantive identity prose. |
| Operational entry points were counted as CLI manual dumps | `COMMAND_RE` counted every backticked command uniformly, and broad headings like `Quick Reference` were always bloat-prone. | Added section-aware command scoring: concise `Essential Commands`, `Operational Entry Points`, `Makefile Entry Points`, `Validation`, `Session Start`, and small `Quick Reference` sections are treated as operational refs, while oversized sections remain bloat. |
| Project-specific stale terms could only be hardcoded in the skill | The stale term list grew organically in the script and could not adapt to unrelated repos without editing the canonical skill. | Added `.xtrm/agent-docs.toml` support for `stale_terms = [...]` or `[audit].stale_terms = [...]`; documented it in audit rules and covered it with a test. |
| `check:gitnexus-no-counter` failed during closeout | Running GitNexus earlier regenerated volatile symbol/relationship/flow counters into tracked `AGENTS.md` and `CLAUDE.md`. | Removed the counters from both docs and reran `npm run check:gitnexus-no-counter` successfully. This reinforces the earlier same-day fix that canonical templates/analyze calls must not emit counters. |
