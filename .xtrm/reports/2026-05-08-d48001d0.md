# Session Report — 2026-05-08

## Summary

The shipped `xt init` / `xt update` Pi runtime path now installs `npm:pi-mcp-adapter` as a required managed Pi package, fixing the startup blocker where `.pi/mcp.json` invoked `npx -y pi-mcp-adapter` for remote MCP servers but the active runtime sync never installed the adapter. The implementation intentionally keeps `pi-mcp-adapter` out of the xtrm-managed extension registry because it is an MCP bridge package, not a Pi extension, and preserves the existing stale-override cleanup for `~/.pi/agent/extensions/pi-mcp-adapter`. The fix landed in `cli/src/core/pi-runtime.ts`, was validated with the targeted Pi runtime safeguards test, recorded in bd memory, closed as `xtrm-n83y`, pushed to `main`, and reflected in `[Unreleased]` for downstream consumers.
