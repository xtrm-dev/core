## Code Changes

### `serena-pool` Pi extension

- Added `packages/pi-extensions/extensions/serena-pool/` plus its registry shim so Pi can manage a shared Serena MCP daemon per git repo root instead of launching duplicate servers per Pi session.
- Deterministic repo-root-to-port mapping and `SERENA_MCP_PORT` wiring let `pi-serena-tools` reuse the daemon across sessions.
- Follow-up cleanup records ownership metadata under `/tmp/serena-pool` and reaps only process groups that the extension owns after the recorded daemon is verifiably dead; unrelated editor/test/hook LSP processes are left alone.
- Added `DEBUG=serena-pool` tracing and an e2e driver under `extensions/serena-pool/test/e2e.ts` for startup/cleanup validation.
- Final v0.7.25 change made debug logging semgrep-safe by using a literal console prefix and passing `msg` as data.

### xtrm-ui compact row marker

- `packages/pi-extensions/extensions/xtrm-ui/format.ts` now exports `TOOL_ROW_MARKER = "›"` and uses it in `renderToolSummary()`.
- `packages/pi-extensions/extensions/xtrm-ui/index.ts` imports the marker for xtrm-ui-owned compact summaries, keeps non-target glyphs untouched, and accepts both `•` and `›` in `highlightExternalToolBadge()` so cached/older compact rows still render badge chrome.
- `EXTERNAL_TOOL_FRAME_PATCH_VERSION` is now `12`, forcing Pi `/reload` to replace stale external-renderer wrappers from earlier experiments.
- `cli/test/xtrm-ui-format.test.ts` was updated for the marker and compatibility behavior; focused vitest coverage is green (`50/50`).
