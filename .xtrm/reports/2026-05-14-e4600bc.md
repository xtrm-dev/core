# Session Report — 2026-05-14

## Summary

Diagnostic-only session. Triaged a regression in `@jaggerxtrm/specialists` v3.15.0 where `sp run --background` silently dies after `payload_breakdown` (seq=4) and never reaches the first agent turn — the operator reported the symptom; this session traced provenance, reproduced the failure cleanly, isolated the root cause, and filed an upstream P0 bead with a regression-catching acceptance criterion. **Root cause:** commit `fa00c5ed` ("fix: stabilize background run integration", `dev/specialists`, May 14 02:46Z) wrapped the inner tmux command as `/bin/bash -lc 'cd <cwd> && exec ${cmd}'`. The `-lc` (login + non-interactive) forces bash to rebuild PATH from `/etc/profile` + `~/.bash_profile` only — `~/.bashrc` (where NVM init puts `pi` on PATH on this and most setups) is **not** sourced. Result: `spawn('pi', args, ...)` in `src/pi/session.ts:669` throws `ENOENT` mid-supervisor; tmux session terminates within milliseconds; obs DB has exactly 4 events per failed run. **Provenance:** the change was bundled into the same commit that closed `unitAI-sxmmy` (P2, test fix — Supervisor's DB bootstrap) and `unitAI-dq6vr` (P3, cosmetic "job ID not yet available" warning race). The `-lc` was a side effect of the wrapper hardening (intent: disambiguate `exec cd ...` parsing as the `cd` builtin) — but `-c` alone would have sufficed; `-l` was the unnecessary addition. **Gates that missed it:** the integration test (`tests/integration/cli/run.integration.test.ts -t background`) only verifies handoff file + readStatus(), never an actual agent turn. Reviewer audit on `unitAI-sxmmy` returned PASS 91 with that same blind spot. CI inherited the gap. CHANGELOG entry talked about fixing the cosmetic warning, not the wrapper change. Filed `unitAI-baz0t` (P0, specialists) with the full root-cause analysis, three fix paths (revert `-lc` → just `-c` / resolve `pi` to absolute / forward PATH via tmux `-e`), and a proposed acceptance criterion: `sp run --background` must (a) write `turn:start` (seq ≥ 5) to obs DB within 30s AND (b) keep an `sp-*` tmux session visible — both together. Zero xtrm-tools-side code changes this session.
