## Code Changes

### CLI bd patching

- `cli/src/core/bd-auto-stage-patch.ts` centralizes the bd patch. It parses `.beads/config.yaml`, flips `export.git-add` to `false`, resolves the active pre-commit path from `core.hooksPath` or `.git/hooks`, appends a clearly marked XTRM shim containing `git add -f .beads/issues.jsonl 2>/dev/null || true`, and is idempotent.
- The hook resolver accepts both default `.git/hooks/pre-commit` and bd v1.0.3 `.beads/hooks/pre-commit`; it only reports `.beads/hooks` as misconfigured when `core.hooksPath` points there but the hook file is absent.

### CLI dependency maintenance

- `cli/src/core/dependency-maintenance.ts` checks installed bd and GitNexus versions against npm latest, flags major upgrades for operator confirmation, attempts non-major/missing upgrades on apply, runs `bd doctor --fix --yes`, and runs `gitnexus analyze` when status is stale/not-indexed/missing/schema-drifted.
- `xt init` now prints a dependency-maintenance summary after project bootstrap and before verification.

### `xt update` and sweep mode

- `cli/src/commands/update.ts` now includes bd auto-stage patch state in drift detection, applies it independently of registry drift, includes maintenance summaries in JSON/table output, and adds `--all-repos` to scan `~/dev` and `~/projects`.
- `--all-repos --apply` commits changed repos individually with `chore: apply bd auto-stage patch (xtrm-tools auto-applied)` after each successful patch/update.

### Pi compact UI

- `packages/pi-extensions/extensions/xtrm-ui/format.ts` raises compact subject/meta limits to 80/64 and adds `joinCompactMeta()` plus `formatPayloadSize()`.
- `packages/pi-extensions/extensions/xtrm-ui/index.ts` uses `bash:<command>` for native bash pending/final rows, includes payload size in native/external compact summaries, and keeps expanded-output behavior intact through the existing `xtrmOriginalText` details path.
- `cli/test/xtrm-ui-format.test.ts` covers compact metadata joining and payload-size formatting.

### Tests, build, and release surface

- `cli/src/tests/bd-auto-stage-patch.test.ts` covers dry-run, apply idempotency, relative `core.hooksPath`, valid `.beads/hooks`, and missing `.beads/hooks/pre-commit` warning behavior.
- `cli/src/tests/update.test.ts` covers update-only patch behavior and `--all-repos` help exposure.
- `cli/dist/index.cjs` and map were rebuilt via root `npm run build` for the CLI work.
- `package.json`, `cli/package.json`, `packages/pi-extensions/package.json`, and `package-lock.json` are prepared at `0.8.1`.
