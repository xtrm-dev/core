## Overview

`xtrm-ui` is a Pi extension that wraps the visual chrome from
[pi-dex](https://github.com/mariozechner/lemmy/tree/main/packages/pi-dex) with
XTRM-specific defaults. It provides the pidex header, themes, and tool expansion
settings — but deliberately **omits the footer**, letting `custom-footer` own that
slot exclusively.

## Credits

Themes and header layout are derived from **pi-dex** by
[@mariozechner](https://github.com/mariozechner). Diff preview behavior is inspired
by **@heyhuynhgiabuu/pi-diff**.

`xtrm-ui` keeps a compatibility-safe adapter path because `@heyhuynhgiabuu/pi-diff`
currently ships extension-first sources without a stable library export surface.
To avoid runtime breakage across Pi versions, xtrm-ui vendors a local renderer that
matches pi-diff style (line numbers + inline change emphasis) while keeping
xtrm-ui as the only write/edit wrapper.

## Why Not pi-dex Directly

pi-dex calls `setFooter()` unconditionally, which races with `custom-footer`'s
`setFooter()` call — whichever fires last wins, causing the statusline to flicker
or revert to the stock pi-dex footer. `xtrm-ui` sets `showFooter: false` by default
and never calls `setFooter()`, giving `custom-footer` stable ownership of the
statusline.

## Diff Rendering Integration

- `edit` tool expanded view now renders richer unified diff output with:
  - old/new line number gutters
  - paired +/- line highlighting
  - word-level emphasis for changed segments
- `write` tool now distinguishes:
  - created file (`line` summary)
  - unchanged write (`no changes`)
  - overwrite/update (`+/-` stats + expanded rich diff preview)
- Footer/theme ownership behavior is unchanged:
  - `showFooter: false` remains default
  - custom-footer remains the sole footer owner

## External Tool Chrome

Non-native Pi tools (MCP tools and extension tools such as `structured_return`,
`process`, Serena, and GitNexus) are compacted and visually distinguished by
`xtrm-ui` so they do not dominate compact sessions. The renderer preserves the
original tool result in `details.xtrmOriginalText` so Ctrl+O expanded mode can
show the full output even when collapsed mode uses a one-line summary.

Two chrome modes are available:

| Mode | Behavior | Use when |
|------|----------|----------|
| `background` | Native-density single-row summaries with a subtle cold full-row tint and brighter tool-name badge | Default; best for dense mixed native/MCP sessions |
| `box` | Tight cold-colored box around compact content; expanded output uses the full box | Useful when non-native tool output needs stronger visual separation |

Known tool families receive tailored compact summaries and color families:

- Serena MCP tools (`find_symbol`, `read_file`, `search_for_pattern`, etc.)
- GitNexus MCP tools (`gitnexus_query`, `gitnexus_context`, `gitnexus_impact`, etc.)
- `structured_return` command results
- `process` background-process manager results
- generic external Pi tools as a fallback

The implementation patches Pi's `ToolExecutionComponent` fail-open and targets
the live Pi package path before falling back to legacy package names. This avoids
patching stale local package copies when Pi is installed as
`@earendil-works/pi-coding-agent`.


## Defaults

| Preference | Default | Notes |
|------------|---------|-------|
| `themeName` | `pidex-dark` | Switch with `/xtrm-ui-theme dark` |
| `density` | `compact` | Switch with `/xtrm-ui-density comfortable` |
| `showHeader` | `true` | Toggle with `/xtrm-ui-header off` |
| `compactTools` | `true` | Tool blocks collapsed by default |
| `showFooter` | `false` | Intentionally off — custom-footer handles this |
| `compactExternalToolResults` | `true` | Summarize non-native tool results in compact mode |
| `externalToolChrome` | `background` | Switch with `/xtrm-ui chrome background|box` or `/xtrm-ui-external-chrome` |
| `toolRowBg` | `false` | Optional native tool-row background |

Preferences are persisted per session via `pi.appendEntry` and restored on
`session_start` / `session_switch` / `session_fork`.

## Commands

| Command | Description |
|---------|-------------|
| `/xtrm-ui` | Show current preferences and context usage; also accepts `chrome background|box` |
| `/xtrm-ui-theme dark\|light` | Switch theme |
| `/xtrm-ui-density compact\|comfortable` | Switch density |
| `/xtrm-ui-header on\|off` | Toggle header |
| `/xtrm-ui-external-chrome background\|box` | Switch non-native tool chrome |
| `/xtrm-ui-compact-tools on\|off` | Toggle external tool result compaction |
| `/xtrm-ui-rowbg on\|off` | Toggle native tool-row background |
| `/xtrm-ui-forcetheme on\|off` | Control whether xtrm-ui owns the active theme |
| `/xtrm-ui-reset` | Restore all defaults |

## Install

`xtrm-ui` is a managed extension using the **global symlink model**:

- Source: `packages/pi-extensions/extensions/xtrm-ui/` (git-tracked)
- Global symlink: `~/.pi/agent/extensions/xtrm-ui` → `packages/pi-extensions/extensions/xtrm-ui/`

On `xtrm install` (or `xt pi` / `xt attach`), the symlink is created/verified.
Pi auto-discovers extensions in `~/.pi/agent/extensions/`, so no manual
`pi install -l` step is required.

pi-dex (`npm:pi-dex`) is **not** installed when using xtrm-ui — it was removed
from managed packages to avoid the footer conflict.

`xt pi reload` also prunes stale `npm:pi-dex` entries from Pi settings when it finds
an older machine still loading pi-dex alongside xtrm-ui.

`@heyhuynhgiabuu/pi-diff` is declared as an extension dependency for capability
parity/reference. Rendering currently uses the local compatibility adapter described
above until a stable library API is exported upstream.
